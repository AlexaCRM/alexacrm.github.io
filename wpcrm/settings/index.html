On this page we're going to walk through all the settings available to you in Integration for Dynamics CRM Plugin. You'll find information about general CRM Plugin settings as well as Connection, Login, Roles Mapping and Forms settings. Please note this documentation is relevant for version 0.9.13 and later.

To configure the plugin go to WordPress Admin > Dynamics CRM. Then walk through the documentation below in each section to get information on every setting.

Connection settings

On this tab you have some simple and basic settings for establishing connection to Dynamics CRM.

Microsoft Dynamics CRM supports two security models for authentication: claims-based authentication and Active Directory authentication. The type of authentication used depends on the type of deployment your application is accessing: Microsoft Dynamics CRM Online or Microsoft Dynamics CRM 2015. CRM Plugin supports both CRM Online (Deployment type: Office 365) and CRM on-premises deployment.

Deployment type
Internet Facing (On-premises) or CRM Online (Office 365). Type of deployment if your CRM hosted on Microsoft services (contoso.crm5.dynamics.com) it's includes word "dynamics" then your deployment type will be "Office 365". If CRM hosts on your domain then choose "On-premises".
CRM Server Url
Absolute url to your Microsoft Dynamics CRM, https://contoso.crm.dynamics.com for Dynamics CRM online or https://contoso.crm2015.net for on-premises deployment.
User Name
CRM systemuser login name, this user will be used to authenticate and access the CRM data, be sure that user have permissions to access Dynamics CRM.
Password
CRM systemuser password, required to authenticate to CRM.
